Do's & Don'ts

Do

  • Keep the aura to its four colors in fixed order (blue → violet → pink → amber).
  • Limit each view to one active aura and one primary button.
  • Build the aura with the frame + solid-inner-layer technique; respect reduced-motion.
  • Use core blue as the only interactive accent; reserve copper for status badges.
  • Express depth through the stone ramp and 1px borders — keep surfaces matte.
  • Use JetBrains Mono only for stack badges and code.

Don't

  • Put the aura on nav, footer, secondary buttons, inputs, or body text.
  • Run two auras — or leave the featured card's aura on at rest.
  • Use radius-full on cards/containers, or push type weight past 600.
  • Scatter the aura colors as a general palette or reorder the stops.
  • Use the mask-composite / @property border-clip approach — it fails in some engines.
  • Use pure black (#000) for surfaces; the base is stone-950.
